The Live Dealer Technology Stack

Core Components

  1. Game Control Unit (GCU): Hardware encoder about the size of a shoebox attached to each gaming table. Encodes video feed and connects to studio network.
  2. Camera Systems: Multiple HD/4K cameras (typically 3-7 per table) capture different angles including overhead, player POV, and close-ups.
  3. OCR Technology: Optical character recognition reads cards, dice, roulette wheel results in real-time.
  4. Streaming Servers: Content delivery network (CDN) infrastructure distributes video to thousands of players simultaneously.
  5. Game Software: Manages bets, player interaction, dealer communication, and game flow.
  6. RFID/Sensors: Embedded in chips, cards, and tables to track game state.

The Game Control Unit (GCU)

The GCU is the heart of live dealer technology. Every gaming table has a dedicated GCU that performs these functions:

  • Video Encoding: Converts raw camera feeds into compressed streams (H.264/H.265 codec)
  • Data Integration: Combines video with game data (card values, wheel results, etc.)
  • Encryption: Secures stream to prevent hijacking or tampering
  • Synchronization: Ensures audio, video, and data arrive together
  • Quality Management: Adjusts bitrate based on network conditions
Technical Specs: Professional GCUs typically encode at 2,000-8,000 kbps for HD (1080p) and 10,000-15,000 kbps for 4K streaming. Latency is kept under 500ms for optimal player experience.

Multi-Camera System Architecture

Camera Placement

A typical live blackjack table uses 6-8 cameras:

  • Camera 1-2: Wide angle shots showing dealer and table

Optical Character Recognition (OCR)

How OCR Reads Cards

OCR technology instantly identifies playing cards without human input:

  1. Card Design: Special cards with optimized corner indices (larger, high-contrast symbols)
  2. Camera Capture: HD close-up camera focused on card corner
  3. Image Processing: Software isolates card corner, removes background noise
  4. Pattern Matching: Compares captured image against database of all 52 cards
  5. Confidence Score: Algorithm assigns probability (95%+ required for accuracy)
  6. Data Transmission: Card value sent to game server in under 100ms

OCR for Roulette

Roulette uses a different OCR approach:

  • High-speed Camera: Captures wheel at 240+ FPS
  • Ball Tracking: AI tracks ball position frame-by-frame
  • Number Recognition: OCR reads winning pocket number
  • Wheel Sensors: Magnetic or optical sensors confirm result
  • Cross-Verification: Visual OCR + sensor data = 99.9%+ accuracy

Error Handling

When OCR confidence is below threshold:

  • Manual Review: Dealer reviews result on tablet
  • Supervisor Verification: Pit boss confirms ambiguous outcomes
  • Multi-Angle Analysis: Secondary cameras provide backup
  • RFID Backup: Chip-embedded cards provide electronic confirmation

Streaming Infrastructure

Content Delivery Network (CDN)

Live streams are distributed via global CDN with edge servers in 100+ locations. Benefits:

  • Reduced Latency: Players connect to nearest server (200-500ms typical)
  • Scalability: Supports 10,000+ concurrent players per table
  • Redundancy: Automatic failover if primary server fails
  • Adaptive Bitrate: Stream quality adjusts to connection speed

Streaming Protocols

  • WebRTC: Ultra-low latency (under 500ms), used for premium tables
  • HLS (HTTP Live Streaming): Standard protocol, 2-3 second latency
  • RTMP (Real-Time Messaging): Legacy protocol, being phased out
